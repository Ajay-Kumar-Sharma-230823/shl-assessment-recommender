# SHL Assessment Recommender

A production-ready conversational AI system that helps HR managers and recruiters find the right SHL assessments from the official product catalog.

## Overview

The system is a stateless FastAPI backend with an AI agent powered by Anthropic Claude. It recommends **only real SHL assessments** from a scraped + FAISS-indexed catalog. Every recommendation includes an exact name and URL from the official SHL catalog — no hallucination.

## Project Structure

```
shl_recommender/
├── main.py          # FastAPI app — GET /health, POST /chat
├── agent.py         # Core agent with scope guard + LLM orchestration
├── retriever.py     # FAISS vector search + catalog lookup
├── scraper.py       # SHL catalog scraper + fallback catalog
├── models.py        # Pydantic request/response models
├── prompts.py       # All system prompts
├── catalog.json     # Scraped catalog data (generated by scraper)
├── catalog.csv      # CSV version (generated by scraper)
├── faiss_index/     # Saved FAISS vector index
│   ├── index.faiss
│   └── index.pkl
├── requirements.txt
├── .env.example
├── render.yaml
└── test_agent.py    # 10 test cases
```

## Setup

### 1. Clone and enter directory
```bash
git clone <repo-url>
cd SHL/shl_recommender
```

### 2. Create virtual environment
```bash
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS/Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Set API keys
```bash
copy .env.example .env      # Windows
# cp .env.example .env      # macOS/Linux
```
Edit `.env` and fill in:
```
ANTHROPIC_API_KEY=your_key_here
GEMINI_API_KEY=your_key_here   # optional fallback
```

### 5. Run the scraper (builds catalog + FAISS index)
```bash
python scraper.py
```
This will:
- Scrape Individual Test Solutions from SHL's catalog
- Fall back to a hardcoded catalog of 23 real assessments if scraping fails
- Save `catalog.json`, `catalog.csv`, and `faiss_index/`

### 6. Start the server
```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

## API Usage

### Health Check
```bash
curl http://localhost:8000/health
# → {"status": "ok"}
```

### Chat (stateless — send full history every time)
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "I need to hire a mid-level Java developer"}
    ]
  }'
```

**Multi-turn example:**
```bash
curl -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "I need to hire a Java developer"},
      {"role": "assistant", "content": "{\"reply\": \"What seniority level?\", \"recommendations\": [], \"end_of_conversation\": false}"},
      {"role": "user", "content": "Senior level, 5+ years, needs cognitive and personality tests"}
    ]
  }'
```

**Response schema:**
```json
{
  "reply": "Here are the top SHL assessments for a senior Java developer...",
  "recommendations": [
    {
      "name": "Verify G+ (Global Skills Assessment)",
      "url": "https://www.shl.com/solutions/products/product-catalog/view/verify-g-plus/",
      "test_type": "A"
    }
  ],
  "end_of_conversation": false
}
```

## Testing

```bash
cd shl_recommender
pytest test_agent.py -v
```

Tests:
1. `test_health_endpoint` — GET /health returns 200 + correct body
2. `test_vague_query` — Vague query returns empty recommendations
3. `test_detailed_query` — Detailed query returns 1-10 recommendations
4. `test_schema_compliance` — Every response has required fields
5. `test_catalog_only` — All URLs are from shl.com
6. `test_prompt_injection` — Injection attempts return refusal
7. `test_off_topic` — Non-SHL queries return refusal
8. `test_refinement` — Adding constraints updates recommendations
9. `test_turn_cap` — Service responds within 30 seconds
10. `test_comparison` — Compare query returns grounded answer

## Deployment (Render.com)

1. Push code to GitHub
2. Go to [render.com](https://render.com) → New → Web Service
3. Connect your repository
4. Set root directory to `shl_recommender`
5. Build command: `pip install -r requirements.txt && python scraper.py`
6. Start command: `uvicorn main:app --host 0.0.0.0 --port $PORT`
7. Add environment variables:
   - `ANTHROPIC_API_KEY`
   - `GEMINI_API_KEY` (optional)

Or use the included `render.yaml` with `render deploy`.

## Key Behaviors

| Scenario | Agent Behavior |
|----------|---------------|
| Vague query ("I need an assessment") | Asks ONE clarifying question, no recommendations |
| Detailed query (role + context) | Returns 1-10 catalog-grounded recommendations |
| Prompt injection | Firmly refuses, stays in role |
| Off-topic (salary, legal) | Politely refuses, redirects |
| Refinement ("add personality test") | Updates recommendations, acknowledges change |
| Comparison ("OPQ vs Verify") | Catalog-grounded comparison |
| User satisfied ("perfect, thanks!") | Sets end_of_conversation: true |
| Turn 6+ with no recommendation | Forces recommendation with available context |
